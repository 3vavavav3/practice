1 спецификация программного модуля
1.1 общие сведения:
- наименование: "модуль для учета заявок на ремонт автомобилей"" 
- заказчик: Автосервис «АвтоТранс»
- исполнитель: Сервисный центр «ТехноПолис»
- платформа и стек: ASP.NET Core (C#) + PostgreSQL

1.2 функциональные требования
1.2.1 управление заявками: 
- возможность добавления заявок в бд с указанием параметров: номер заявки, дата добавления, вид авто, модель авто, описание проблемы, ФИО клиента, номер телефона, статус заявки.
- возможность редактирования заявок (этап выполнения, описание проблемы, ответственный за выполнение работ)
- просмотр списка заявок
- получение уведомлений (смена статуса)
1.2.2 поиск:
- по номеру заявки
- по параметрам
1.2.3 расчет статистики (отдел обслуживания):
- количество выполненных заявок
- среднее время выполнения заявки
- статистика по типам неисправностей
1.2.4 назначение ответственных: 
- добавление автомеханика к заявке
- отслеживание состояния работы
- добавление комментариев к заявке автомехаником (автозапчасти, материалы, т.п.)

1.3 нефункциональные требования:
- поддержка windows
- простой интерфейс, понятные уведомления
- быстрота работы
- система авторизации и роли

1.4 входные данные
1.4.1 исходные данные (импорт из файлов):
- заявки: inputDataRequests.* (id, дата начала, тип авто, модель авто, проблема, статус, дата завершения, запчасти, id мастера, id клиента)
- комментарии: inputDataComments.* (id, текст, id мастера, id заявки)
- пользователи: inputDataUsers.* (id, фио, телефон, логин, пароль, тип)
1.4.2 данные, вводимые пользователем:
- при добавлении/редактировании заявки (п.1.4.1)
- при добавлении комментария (текст)
- логин и пароль
- поисковый запрос

1.5 выходные данные
1.5.1 отображаемые экраны:
- главное окно (список заявок)
- формы добавления/редактирования
- просмотр деталей заявки с комментариями
- окно результатов поиска
- окно статистики
1.5.2 системные сообщения:
- предупреждения, подсказки
